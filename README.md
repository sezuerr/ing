# ing MVP 小程序基础架构

这是一个微信原生小程序 + 微信云开发的 MVP 骨架，用于验证基于 LBS 模糊位置的半匿名同城校园社交。

## 导入方式

1. 用微信开发者工具导入当前目录。
2. 在 `app.js` 中把 `globalData.envId` 改成你的云开发环境 ID。
3. 在云开发控制台创建数据库集合，集合清单见 `docs/database.md`。
4. 右键 `cloudfunctions/api` 上传并部署云函数，安装依赖。
5. 可将 `data/universities.seed.json` 和 `data/daily_topics.seed.json` 导入对应集合。

## 已实现

- 5 个自定义 Tab：设置、发现、发布、信息、我的。
- 中央黄色悬浮发布按钮和灯泡品牌视觉。
- 发现页划卡模式、范围筛选、随机话题筛选、点亮、私密回复、举报/不感兴趣。
- 发布页话题 Icon、标题、正文、图片上传、可见度、模糊位置。
- 信息页通知流、会话列表、解除配对、聊天轮询。
- 我的页资料、统计、帖子流、帖子详情评论。
- 统一 `api` 云函数，包含计划中的核心接口和 `getMessages` 聊天读取接口。
- 无云环境时自动使用本地 mock 数据，方便先看 UI。

## 重要约定

- 不长期保存精确经纬度，前端只上传粗粒度 `geoHash`。
- 敏感写操作统一走云函数，前端不直接写核心集合。
- 点亮不可撤销；互亮后创建 `matches` 和 `conversations`。
- 帖子默认 `status=visible`、`reviewStatus=pending`，便于 MVP 演示；正式上线前应接入内容安全审核后再展示。
